# 工程场景 OCR 识别结果评测与错误归因分析报告

生成时间：2026-06-09 16:47

## 1. 核心结论

本次共分析 500 张工程场景图片，覆盖 3000 个字段级识别结果。
OCR 整体字段级准确率为 86.3%，图片级完全正确率为 45.6%，平均置信度为 0.861。
待人工重点复核样本共 235 条，主要集中在低亮度、遮挡、模糊和字体过小等质量问题图片。

## 2. 错误类型分布

- 漏识别：90 次，占 21.9%
- 字段缺失：89 次，占 21.7%
- 数字识别错误：72 次，占 17.5%
- 字母识别错误：63 次，占 15.3%
- 顺序错误：60 次，占 14.6%

## 3. 字段准确率

- 堆场口：83.4%
- 日期：84.8%
- 作业区域：85.0%
- 设备状态：87.4%
- 龙门吊编号：88.0%
- 桥吊编号：89.2%

## 4. 设备类别错误率

- 桥吊：15.2%
- 龙门吊：14.0%
- 堆场口：13.5%
- 混合作业牌：12.2%

## 5. 图片质量影响

不同质量标签下的主要风险为：字体过小 错误率 19.7%、模糊 错误率 18.4%、倾斜 错误率 17.9%、噪声 错误率 17.8%。
这说明 OCR 错误并非均匀分布，而是明显受拍摄质量、文本清晰度和背景干扰影响。

## 6. 复核与优化建议

- 对桥吊编号、龙门吊编号、堆场口、作业区域和日期字段增加格式规则校验。
- 对低亮度、遮挡、模糊、字体过小样本优先进入人工复核队列。
- 采集侧建议控制拍摄角度，补充照明，避免设备结构遮挡编号区域。
- 模型侧建议补充相似字符样本，例如 0/O、1/I、3/8，以及中文状态字段的混淆样本。
- 业务侧可把高风险字段作为安全资料整理的检查项，降低编号录入错误带来的后续追溯成本。
